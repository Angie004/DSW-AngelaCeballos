FRAMEWORKS CSS


Un Frameworks de CSS es una biblioteca de estilos genéricos que se puede usar para implementar diseños web. Tiene en cuenta ciertas utilidades que pueden aprovecharse frecuentemente en el diseño web. Una de las ventajas más importantes de un Frameworks bien diseñado e implementado, es que asegura que va a funcionar en una gran variedad de navegadores y también aporta cierto grado de fiabilidad e eficacia ya que se supone que está muy probado para garantizar que no hay errores.

Uno de los principales objetivos de los Frameworks de CSS es facilitar el desarrollo del diseño de un sitio web ahorrando tiempo ya que proporciona diversas características de estilos para el o los sitios sin necesidad de tener que crearlos/digitarlos; simplemente es como una plantilla de algunas características visuales del sitio que se aplica para ya obtener sus características automáticamente y poder continuar con otras tareas del diseño sin perder tiempo en ello. Adicionalmente, como ya se mencionó, esta plantilla va a garantizar que funciona, por lo que no se tendrá que perder tiempo y esfuerzo atendiendo y resolviendo errores.

Sin embargo, a pesar de tener muchas ventajas, una de las desventajas más notorias es la implementación de un Frameworks que no sea necesario o correcto, lo que podría agregar una gran cantidad de ancho de banda y tiempo de carga innecesarios con elementos que no aportan nada al sitio y que no debería tener. Esto es porque muchas de estas bibliotecas vienen con un paquete, en ocasiones muy grande, de características y muchas de ellas podría no tenerlas nuestro sitio web; por lo que estaríamos añadiendo todas esas cosas innecesarias que solo consumirán recursos, mientras que si se creara algo personalizado para nuestro sitio, tendría lo justo y lo necesario y ese problema no existiría. Por esto se debe tener cuidado al aplicar Frameworks de CSS en un proyecto.

Los Frameworks CSS se clasifican en dos tipos según el uso para el cual estén destinados: Multipropósito y Propósito específico. Los primeros, como su nombre lo dice, se refiere a aquellas bibliotecas que vienen con un conjunto de opciones generales con las características más comunes o requeridas, de manera amplia; mientras que el segundo grupo se refiere a bibliotecas con un objetivo definido y específico, especializada en una función determinada que se requiera.
